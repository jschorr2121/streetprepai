## Stale PRD assumptions — corrections

1. **Migration numbering is wrong.** PRD says migration `0007`; migrations `0000`–`0009`
   are used (`0006`=curriculum, `0007`=qbank_seed, `0009`=perf indexes). Next free
   number is **`0010`** (e.g. `0010_chat_threads.sql`).
2. **The AI SDK is NOT installed.** PRD claims `ai` v6 + `@ai-sdk/anthropic` are
   "already installed (Unit 2) and unused" — `package.json` has neither, only
   `@anthropic-ai/sdk` and `openai`. Issue 01 starts with a dependency add
   (`ai`, `@ai-sdk/anthropic`, `@ai-sdk/react`) and a docs check against the actual
   installed v6 API (Context7). Bundle-vigilance rule applies (>30kb needs
   justification — this one has it, but say so in the PR).
3. **`search_chat_logs` is NOT semantic.** It's lowercase substring matching
   (`assistant-tools.ts` ~187-223). The real semantic path
   (`lib/data/semantic-recall.ts#findSimilarChats`, pgvector RPC
   `match_chat_embeddings`) is a separate module the tool doesn't use. Issues 02/04
   inherit the false "already semantic" premise — DECIDE explicitly whether to swap
   the tool onto `findSimilarChats` (embeddings now flow: chats are persisted and
   embedded as of session 3) or keep keyword search. That decision cascades into 04.
4. **`CHAT_SYSTEM` is guide-scoped** ("using ONLY the content of the guide") — not
   reusable as the standalone assistant's system prompt. Write a new prompt.
