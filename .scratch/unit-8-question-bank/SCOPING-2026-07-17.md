## Stale PRD assumptions — corrections

1. **Migration numbering/naming:** PRD says `0006_qbank.sql`; real file is `0006_curriculum.sql` (combined scope). `0000`–`0010` are all taken (`0010` = chat_threads, Unit 9). **Next free number: `0011`.**
2. **No `rubric jsonb` column.** Real schema splits it: `key_points jsonb` + `misconceptions jsonb` + `model_answer text`, plus a `question_type` CHECK the PRD never mentions. Anything referencing "the rubric column" must use those.
3. **No exported `correct = score ≥ 7` constant.** Scoring is 0–100 and the model returns `correct: boolean` in the forced `save_grade` tool call (rubric in `GRADING_SYSTEM`); there is no numeric gate in app code to reuse.
4. **`qbankGradingLimiter` is fail-closed** (`onStoreError: "deny"`) — a deliberate Phase-1 security decision, tested both ways in `limiters.test.ts`. Do not switch it to degrade-open.
5. **`lib/data/interview-questions.ts` was never migrated into the DB.** The 532-question corpus was freshly authored by content agents from `extra_content/` PDFs; `interview-questions.ts` still solely powers mock-studio — correctly independent, wrong PRD rationale.
6. **Dashboard weak-areas widget already exists**, contradicting the PRD's "later dashboard unit".
7. **Module split differs:** grading is `lib/ai/grading.ts`, shared 4-ways (question bank, section drills, chapter gates, daily drill) because chapters/sections/gates exist now.
